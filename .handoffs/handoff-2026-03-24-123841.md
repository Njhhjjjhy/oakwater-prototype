## Problems encountered and how they were solved

1. **No git repo**: Project had no git history, making reverts impossible. When user asked to revert, had to rewrite from memory. Lesson: should have initialized git early.

2. **100vh bug on iOS Safari**: The `100vh` is taller than the visible viewport on iOS because it includes the address bar. Fixed with `100dvh` (dynamic viewport height) and JS fallback setting `--vh` custom property.

3. **Body scroll lock on iOS**: Simple `overflow: hidden` on body doesn't prevent scroll on iOS Safari. Fixed with `position: fixed; top: -{scrollY}px; width: 100%` pattern, restoring scroll position on unlock.

4. **Logo white background**: Original logo PNG had white background that clashed with sand site background. Fixed with ImageMagick: `-fuzz 15% -transparent white -trim +repage`.

5. **WCAG contrast failures**: Multiple color pairs failed 4.5:1 ratio. Most were low-opacity text on backgrounds. Fixed by raising opacity values and lightening the teal-dark variant from #1B706A to #4B8E89.

6. **Layout property animation**: `padding-left` was being animated on hover, causing reflow. Changed to `transform: translateX()` which is GPU-composited.

7. **Heading hierarchy**: Jumped from h1 to h3 with no h2. Section labels were `<p>` tags — changed to `<h2>` to create proper h1 > h2 > h3 flow.

8. **Double 'w' in logo filename**: Original file was `oakwwater-logo.png` (typo). Created clean version as `oakwater-logo-clean.webp` with correct spelling.

## Architecture and design notes

### Single-file architecture
The entire site is one `index.html` with embedded `<style>` and `<script>`. No build tools, no framework, no dependencies other than Google Fonts. This was a deliberate choice from the original prompt.

### Design system (CSS custom properties)
All colors, fonts defined as CSS variables on `:root`:
- Colors: `--sand`, `--ink` (with opacity variants -70, -40, -20, -10, -06), `--teal`, `--teal-dark`, `--teal-10`, `--teal-15`, `--white`
- Fonts: `--serif` (Cormorant Garamond), `--sans` (Raleway), `--mono` (IBM Plex Mono)

### Typography hierarchy
- **h1**: Cormorant Garamond, weight 300, clamp(2.8rem, 5vw, 3.75rem), teal color
- **h2** (section labels): IBM Plex Mono, 0.75rem, teal, used as semantic headings styled small
- **h3**: Cormorant Garamond, 1.5rem, weight 500
- **Body**: Raleway, 0.9-1rem, ink-70

### Section pattern
Sections alternate between sand background (direct on body) and white background (wrapped in `.who` or `.invest` containers with 12px border-radius). The risk section uses dark `--ink` background with rounded corners. Footer is dark with top border-radius only.

### Team accordion
Uses `grid-template-rows: 0fr → 1fr` transition for height animation (no `max-height` hack). Each `.team-member` is a border-separated row with `.team-header` (role=button, tabindex=0, aria-expanded) and `.team-body` containing the bio. Only one member can be open at a time.

### iOS scroll lock
Uses `position: fixed` on body with stored scroll position, not `overflow: hidden`, because iOS Safari doesn't respect overflow hidden on body for scroll prevention.

### Image strategy
All team photos converted from original jpg/png/jpeg to webp at quality 85. Logo converted at quality 95 with alpha preservation after white background removal. All team photos are `loading="lazy"`. Photos display as 56px circles (48px on mobile) with CSS grayscale filter that transitions to full color on hover/open.
